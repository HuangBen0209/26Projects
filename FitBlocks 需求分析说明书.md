# FitBlock 健身小游戏功能需求文档

## 一、文档概述

### 1.1 文档目的

本功能需求文档明确基于计算机视觉的**FitBlock**健身小游戏的核心功能、非功能要求、用户场景及验收标准，作为项目开发、测试、验收的核心依据，确保产品贴合 “AI 体感 + 俄罗斯方块 + 健身” 的核心定位，兼顾娱乐性与健身实用性。

### 1.2 项目名称

FitBlock（中文名：体趣方块）

### 1.3 产品定位

以经典俄罗斯方块为载体，通过计算机视觉识别用户手臂动作实现无接触控制，将娱乐游戏与上肢健身深度融合，打造低门槛、高沉浸、碎片化的居家轻量健身体验，适用于学生、办公族、减脂人群等全年龄段用户。

## 二、核心用户与场景

### 2.1 核心用户



1. 健身新手：无专业健身基础，希望通过趣味方式入门运动，避免枯燥

2. 办公族：久坐肩颈不适，利用碎片化时间（10 分钟内）做轻量上肢运动

3. 学生群体：追求趣味学习 / 娱乐结合，适合校园场景轻量互动

4. 康复辅助人群（轻度）：需上肢轻量康复训练，借助游戏提升训练依从性

### 2.2 核心场景



1. 居家场景：用户居家办公 / 休息时，打开电脑 / 手机，通过摄像头开启游戏，边玩边完成上肢运动

2. 校园场景：班级 / 社团活动中，多人联机对战，增强互动性与趣味性

3. 康复场景：康复师指导下，借助游戏完成标准化上肢康复训练，实时监测动作完成度

## 四、非功能需求

### 4.1 性能需求



1. 运行流畅性：PC 端（配置≥i5 8 代 + 16G 内存 + RTX 2060）帧率≥60FPS，无明显卡顿；手机端（安卓 10.0+，4G 内存以上）帧率≥30FPS

2. 识别性能：姿态识别帧率≥30FPS，延迟≤100ms；正常光线（100-500lux）下识别准确率≥95%，弱光 / 强光环境下≥85%

3. 资源占用：PC 端 CPU 占用≤30%，内存占用≤1G；手机端 CPU 占用≤25%，电量消耗≤15%/ 小时

4. 联机性能：互联网联机延迟≤300ms，掉线率≤1%，重连时间≤5s

### 4.2 兼容性需求



1. 设备兼容：

* PC 端：支持 Windows 10 及以上系统（64 位），兼容普通 USB 摄像头、手机摄像头（通过投屏 / 软件连接）

* 手机端：支持安卓 10.0 及以上、iOS 14.0 及以上系统，兼容主流品牌手机（华为、小米、苹果、OPPO 等）

1. 浏览器兼容（Web 端）：兼容 Chrome 90+、Edge 90+、Firefox 88 + 等主流浏览器，不支持 IE 浏览器

2. 摄像头兼容：兼容 90% 以上主流品牌摄像头（罗技、小米、华为等），无需额外驱动

### 4.3 易用性需求



1. 操作门槛：新手用户无需教程即可在 5 分钟内掌握核心操作（含动作校准）

2. 界面设计：布局简洁，核心按钮（开始、切换、暂停、查看报告）醒目，字体大小适中（≥14px），颜色对比度≥4.5:1

3. 帮助引导：提供新手引导流程（3 步快速上手：校准摄像头→选择模式→开始游戏），内置常见问题解答（FAQ）

### 4.4 安全性需求



1. 数据安全：用户运动数据、个人信息（昵称、头像）加密存储，不泄露给第三方

2. 隐私保护：摄像头权限仅用于姿态识别，不存储、不上传用户面部 / 身体隐私数据，用户可随时关闭权限

3. 付费安全：付费道具 / 会员通过官方渠道支付，支持微信、支付宝、苹果内购，支付流程加密，保障资金安全

### 4.5 可维护性需求



1. 代码结构：模块化设计，核心功能（游戏逻辑、姿态识别、数据统计）独立拆分，便于后续迭代维护

2. 日志记录：完整记录用户操作日志、错误日志，便于问题排查，日志保留时间≥30 天

3. 版本更新：支持在线热更新，核心功能更新无需用户重新下载，重大版本更新提供清晰更新说明

## 五、验收标准

### 5.1 功能验收



1. 核心功能全覆盖：所有 3.1-3.4 节功能均正常实现，无缺失

2. 功能有效性：每个功能均通过实际测试验证，达到 “验收标准” 中对应的指标

3. 逻辑正确性：功能间无冲突，核心流程（校准→开始游戏→过程监测→结束报告）顺畅无 BUG

### 5.2 非功能验收



1. 性能达标：帧率、延迟、资源占用等指标均符合 4.1 节要求

2. 兼容正常：在指定设备、系统、浏览器中均能正常运行，无崩溃、功能异常

3. 易用性达标：新手引导有效，用户操作反馈良好，无复杂操作流程

4. 安全合规：数据加密、隐私保护、付费安全等要求均满足，无安全漏洞

### 5.3 验收流程



1. 功能测试：逐一测试各功能模块，记录测试结果，确保无 BUG

2. 性能测试：使用专业工具测试帧率、延迟、资源占用，验证是否达标

3. 兼容性测试：在不同设备、系统、浏览器中运行，验证兼容性

4. 用户体验测试：邀请 10 + 不同类型用户（新手、办公族、学生）体验，收集反馈，优化细节

## 六、附录

### 6.1 术语解释



1. 计算机视觉：通过摄像头捕捉图像 / 视频，实现对人体动作、姿态的识别与分析

4. 连击：连续消除 2 行及以上的游戏操作，对应额外加分机制
